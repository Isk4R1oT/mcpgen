# External Integrations

## API & Data Sources

### OpenAPI/Swagger Spec Fetching
- **Type**: HTTP-based API documentation ingestion
- **Location**: `backend/services/spec_fetcher.py`
- **Mechanism**:
  - Fetches OpenAPI/Swagger specs from user-provided URLs via httpx (async HTTP client)
  - Auto-discovers spec location from Swagger UI HTML pages (pattern matching)
  - Supports JSON and YAML OpenAPI formats
  - Falls back to HTML content extraction via trafilatura library
  - Detects spec type: openapi_json, openapi_yaml, html, or text
- **Common Discovery Paths**: `/openapi.json`, `/swagger.json`, `/api/openapi.json`, `/api-docs.json`
- **Protocol**: HTTP/HTTPS
- **Timeout**: 30 seconds for initial request, 15 seconds per spec fetch attempt

### PDF & Markdown Documentation Extraction
- **Type**: Local file parsing
- **Location**: `backend/services/spec_fetcher.py`
- **Libraries**:
  - **pdfplumber** - PDF text extraction
  - **trafilatura** - Web content extraction (HTML to text)
- **Use Case**: Parse API documentation from non-standard formats

## LLM & AI Services

### OpenRouter API
- **Provider**: OpenRouter (LLM aggregator)
- **Authentication**: API key via `OPENROUTER_API_KEY` environment variable
- **Integration**: PydanticAI models with OpenRouter provider
- **Location**:
  - `backend/agents/analyzer_agent.py` - API endpoint analysis
  - `backend/agents/generator_agent.py` - Python code generation for MCP servers
  - `backend/agents/chat_agent.py` - Multi-turn conversation
  - `backend/agents/configurator_agent.py` - Configuration recommendations
  - `backend/agents/debugger_agent.py` - Error analysis and fixes
  - `backend/agents/extractor_agent.py` - Spec extraction from unstructured docs
  - `backend/agents/tester_agent.py` - Generated server validation
- **Default Model**: `x-ai/grok-code-fast-1` (from `.env.example`) or `anthropic/claude-sonnet-4-5`
- **Endpoint**: `https://openrouter.ai/api/v1` (as configured in `backend/eval/openrouter_judge.py`)
- **Capabilities**:
  - Structured JSON output (response_format: json_object)
  - Tool use (via PydanticAI)
  - Multiple concurrent requests (async)

### DeepEval LLM-as-Judge
- **Type**: Testing & evaluation framework
- **Location**: `backend/eval/openrouter_judge.py`, `backend/eval/metrics.py`
- **Custom Implementation**: OpenRouterJudge class extends DeepEval's BaseDeepEvalLLM
- **Purpose**: Evaluate quality of generated MCP servers using LLM judgment
- **Uses**: OpenRouter API (same credentials as main agents)
- **Response Format**: JSON schema validation for structured evaluation results
- **Optional**: Included in `pyproject.toml` optional dev dependencies

## Database & Storage

### Supabase (PostgreSQL + Object Storage)
- **Type**: Database and file storage platform
- **Authentication**: API URL + anon key via environment variables
  - `SUPABASE_URL` - Project endpoint (e.g., https://your-project.supabase.co)
  - `SUPABASE_KEY` - Public anon key
- **Location**:
  - `backend/db/client.py` - Singleton client initialization
  - `backend/db/models.py` - Data models (inferred)
  - `backend/db/repositories.py` - CRUD operations
  - `backend/db/store.py` - Storage layer
- **Usage**:
  - Store job metadata (creation date, status, input params)
  - Store generated server source code and Dockerfiles
  - Store execution history and logs
  - User session/auth data (if implemented)
- **SDK**: supabase-py (v2.13+)
- **Access Pattern**: RESTful via supabase-py client

## Containerization & Registry

### Docker Registry Integration
- **Type**: Container image hosting
- **Authentication**: Registry credentials (DOCKER_REGISTRY environment variable)
- **Location**: `backend/services/docker_service.py`
- **Default Registry**: ghcr.io (GitHub Container Registry) - configurable via `DOCKER_REGISTRY`
- **Features**:
  - Build local images via docker-py (`docker.from_env()`)
  - Tag images with registry prefix
  - Push to registry if `DOCKER_REGISTRY_PUSH_ENABLED=true`
- **Mount Point**: `/var/run/docker.sock` (in docker-compose, allows containerized backend to build containers)
- **Output**: Full registry tag format `registry/server-name:tag` returned after successful push

### Docker Daemon
- **Type**: Container runtime
- **Access**: Via docker-py library connecting to local/mounted daemon socket
- **Features Used**:
  - `images.build()` - Compile Dockerfile to image
  - `images.get()` - Retrieve built image
  - `images.push()` - Push tagged image to registry
- **Build Context**: Temporary directory containing:
  - Generated Python server files
  - requirements.txt (pip dependencies)
  - Dockerfile (generated by `backend/pipeline/packager.py`)

## File & Content Processing

### Web Content Extraction (trafilatura)
- **Type**: HTML to text converter
- **Location**: `backend/services/spec_fetcher.py` (line 65, 197)
- **Purpose**: Extract readable text from HTML API documentation pages
- **Use Case**: When Swagger UI spec URL discovery fails, fallback to extracting text content

### PDF Parsing (pdfplumber)
- **Type**: PDF text extraction library
- **Location**: `backend/services/spec_fetcher.py` (lines 179-194)
- **Function**: `extract_text_from_pdf(pdf_bytes: bytes) -> str`
- **Purpose**: Convert user-uploaded PDF documentation to plain text for LLM processing

## HTTP Clients & Networking

### httpx (Async HTTP Client)
- **Location**: `backend/services/spec_fetcher.py`
- **Features**:
  - Async context manager (`AsyncClient`)
  - Automatic redirect following
  - Custom timeouts (30s initial, 15s per fetch)
  - Streaming response handling
- **Use Case**: Fetch OpenAPI specs from URLs, discover Swagger UI locations

### URL Utilities
- **urllib.parse** - URL resolution, base URL extraction
- **Location**: `backend/services/spec_fetcher.py` (lines 127-130)

## Authentication & Security

### Environment-Based Configuration
- **Type**: Pydantic BaseSettings
- **Location**: `backend/config.py`
- **Credentials Managed**:
  - Supabase API key (read-only anon key, not exposed to frontend)
  - OpenRouter API key (never shared, server-side only)
  - Docker registry credentials (not stored, rely on docker CLI auth)
- **Pattern**: All sensitive keys sourced from `.env` file, never hardcoded

### CORS (Cross-Origin Resource Sharing)
- **Location**: `backend/main.py`
- **Allowed Origins**: `http://localhost:5173`, `http://localhost:3000` (dev only)
- **Production**: Requires env-based configuration for security

### Generated Server Auth
- **Pattern**: Auth credentials passed via environment variables to generated Docker containers
- **Design**: Generated MCP servers never expose API keys through tool signatures

## Code Quality & Validation

### OpenAPI Spec Validation
- **Libraries**:
  - prance (schema processing)
  - openapi-spec-validator (validates against OpenAPI standard)
  - openapi-pydantic (converts to Pydantic models for type safety)
- **Location**: `backend/pipeline/parser.py`
- **Purpose**: Ensure user-provided specs are valid before analysis

## MCP Protocol Integration

### FastMCP Server Framework
- **Type**: Python MCP server runtime
- **Location**: Generated in output servers (embedded via Jinja2 template)
- **Version**: >=3.1
- **Transport**: Streamable HTTP (via PydanticAI MCPServerStreamableHTTP in `backend/agents/tester_agent.py`)
- **Purpose**: Runtime for testing generated servers before deployment

## External Service Dependencies (At Runtime)

1. **OpenRouter API** - Required for all agent operations
2. **Supabase** - Required for job persistence
3. **Docker Daemon** - Required for image builds (can be local or remote)
4. **Docker Registry** - Required only if push is enabled
5. **User-provided API URLs** - Required for spec fetching (user provides)
